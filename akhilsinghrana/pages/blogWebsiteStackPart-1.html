Building a Personal Website in 2024: My Tech Stack

In 2024, creating a personal website has become more accessible than ever, thanks to a wide array of powerful tools and technologies. In this blog post, I'll walk you through the tech stack I used to build akhilsinghrana.com and explain why I chose these particular technologies. We'll also explore some alternatives that you might consider for your own projects.

My Tech Stack

Before we dive into the details, here's a quick overview of the technologies I used:

  1. Version Control
    GitHub
  2. Backend
    FastAPI
  3. Frontend
    HTML, CSS, JavaScript, Bootstrap 5
  4. Containerization
    Docker
  5. Deployment (covered in part-2)
    GitHub Actions, Google Cloud Run

Let's break down each component and discuss why I chose it for my 2024 website.

1. Version Control: GitHub

GitHub remains a cornerstone of modern software development in 2024. I use it not only for version control but also for deployment via GitHub Actions (which we'll cover in Part 2 of this blog series).

Why GitHub in 2024?

  • Robust version control
  • Excellent collaboration features
  • Integrated CI/CD pipelines
  • Large community and ecosystem

2. Backend: FastAPI

For the backend of akhilsinghrana.com, I chose FastAPI, a modern, fast (high-performance) Python web framework for building APIs.

Why FastAPI in 2024?

  • High performance: FastAPI is one of the fastest Python frameworks available
  • Easy to use and learn: It's designed to be easy to use and learn, with intuitive APIs
  • Automatic API documentation: FastAPI generates OpenAPI (Swagger) docs automatically
  • Python 3.7+ features: It takes full advantage of Python's type hinting and async features

Image: FastAPI performance comparison (placeholder)

3. Frontend: HTML, CSS, JavaScript, Bootstrap 5

For the frontend, I stuck with the tried-and-true combination of HTML, CSS, and JavaScript, enhanced with Bootstrap 5 for responsive design and pre-built components.

Why this frontend stack in 2024?

  • Universal compatibility: These technologies work across all modern browsers
  • Bootstrap 5: Provides a solid foundation for responsive design and consistent UI components
  • Flexibility: This stack allows for both static and dynamic web pages
  • Performance: Vanilla JavaScript can be very performant when used correctly

4. Containerization: Docker

I use Docker to containerize my website, which provides consistency across different environments and simplifies deployment.

5. Deployment

I'll be covering the deployment process in Part 2 of this blog series, where we'll dive into:

  • Using GitHub Actions for continuous integration and deployment
  • Using Google Cloud Registry for storing Docker images
  • Deploying to Google Cloud Run for a serverless setup

Why This Tech Stack in 2024?

You might be wondering why I chose this particular stack in 2024, given the myriad of options available. Here are my reasons:

  1. Simplicity and Maintainability: This stack is relatively simple, making it easy to maintain and update over time.
  2. Performance: FastAPI on the backend and vanilla JavaScript on the frontend can deliver excellent performance.
  3. Scalability: This stack can easily scale as my needs grow.
  4. Learning Curve: For developers familiar with Python and basic web technologies, this stack has a gentle learning curve.
  5. Cost-Effective: Many of these technologies are free and open-source, keeping costs down.
  6. Containerization: Docker ensures consistency across development and production environments.
  7. Serverless Deployment: Google Cloud Run allows for easy, scalable, and cost-effective deployment.

Alternative Tech Stacks

While this stack works well for my needs, there are several alternatives worth considering in 2024:

  1. Next.js or Gatsby: For a more React-based approach with server-side rendering capabilities.
  2. Vue.js with Nuxt: If you prefer Vue over React, this is a solid alternative.
  3. Django: If you need a more batteries-included approach for larger applications.
  4. Svelte: For a compiled approach to frontend development, offering great performance.
  5. Deno and Fresh: If you want to use JavaScript/TypeScript on both frontend and backend.

The Role of LLMs in Website Development

It's worth noting that in 2024, Large Language Models (LLMs) have become an invaluable tool in the web development process. I utilized LLMs extensively while designing and developing this website. They helped in several ways:

  • Generating initial HTML and CSS templates
  • Debugging and optimizing code
  • Suggesting best practices and modern design patterns
  • Answering specific questions about implementation details

While LLMs don't replace the need for solid coding skills and decision-making, they significantly speed up the development process and can help developers learn new techniques on the fly.

Conclusion

Building a personal website in 2024 offers more options than ever before. The stack I've chosen - GitHub, FastAPI, classic frontend technologies, Docker, and Google Cloud - provides a balance of performance, simplicity, flexibility, and scalability that suits my needs perfectly.
